## 3. What was changed — by cluster, with file:line refs and rationale

### 3.1 Cluster 1: Persona consistency across background agents

**Problem:** SOUL.md is only loaded for the user-facing AIAgent. Background agents (memory flush, compression, /btw, scheduled compression, emergency rerun) were creating their own AIAgent instances without the persona, leaking the default Nous Research identity into MEMORY.md / user profile writes, which then poisoned future sessions.

**Fix:** Added `_ephemeral_system_prompt` attribute to GatewayRunner, gated behind the implicit presence check `getattr(self, "_ephemeral_system_prompt", "") or None` so test builders using `object.__new__(GatewayRunner)` don't AttributeError. Passed into every background AIAgent site:

| Site | File:line | Purpose |
|---|---|---|
| Memory flush | `gateway/run.py:728` | Prevents persona drift on memory extraction |
| Context compression (`/compress auto`) | `gateway/run.py:3440` | Persona survives compression |
| Deferred background task | `gateway/run.py:5364` | Persona in emergency rerun |
| `/btw` scheduled compression | `gateway/run.py:5545` | Also dropped `skip_context_files=True` — let SOUL.md load |
| Manual `/compress` | `gateway/run.py:5894` | Same rationale |

**Why not a feature flag:** earlier plan had `HERMES_BACKGROUND_PERSONA=1` as a safety net; post-verification the flag was dropped since the change is backward-compatible (empty string → None → no-op override).

### 3.2 Cluster 2: Session persistence

**#8038 — silent flush-index poisoning.** In `run_agent.py:2301-2347`, the per-session flush cursor `_last_flushed_db_idx` was advanced unconditionally after a batch, even if individual `append_message` calls inside raised. Subsequent flushes skipped those messages — conversations were silently losing history. Fix: advance the cursor inside the per-message try block, only after `append_message` succeeds. Outer exception logs and continues to next flush cycle.

**#6481 — session loss on CLI exit / SIGHUP.** `cli.py` had no atexit or cleanup for `_persist_session`. On SIGTERM or KeyboardInterrupt the in-memory conversation was lost even though the session DB existed. Fix: added `_run_cleanup()` registered via `atexit` that calls `_active_agent_ref._persist_session(history, history)` as a final flush. Guarded against None and AttributeError.

**#9220 — `/sethome` lost on restart.** Home channel was only held in-process; restart wiped it. Fix: at `gateway/run.py:4788`, the `/sethome` command now calls `save_env_value(env_key, str(chat_id))` writing to the user's `.env`. Gateway reads the env var at boot. Roundtrip tested.

### 3.3 Cluster 3: Display leakage generalisation

**Problem:** Hardcoded `if source.platform != Platform.WHATSAPP` guards in `gateway/run.py` for reasoning display. Added more platforms (Signal, BlueBubbles) and the same leak appeared.

**Fix:** `gateway/run.py:3642` — replaced platform-specific guard with tier-driven `if _show_reasoning_effective and response:`. WhatsApp ends up in the quiet tier (`_TIER_LOW`) which sets `show_reasoning=False`. Adding a new platform to the quiet tier is now a one-line change in `gateway/display_config.py`.

### 3.4 Cluster 4: Duplicate replies (live ops fix)

**Diagnosis:** Two concurrent gateways both bound to `+27686284494`:
1. `hermes-gateway.service` (new)
2. `openclaw-gateway.service` (old, had `Restart=always` + `enabled`)

**Fix:** `systemctl --user stop openclaw-gateway.service && systemctl --user disable openclaw-gateway.service`.

**Secondary cause — interim messages.** Even after stopping OpenClaw, single-sender duplicates persisted because `config.yaml: display.interim_assistant_messages: true` was sending mid-turn commentary as a separate message, paired with the final assistant reply — two messages, different formats, looked like a duplicate to Theo. **Fix:** set `display.interim_assistant_messages: false` in `config.yaml`.

### 3.5 Cluster 5: WhatsApp formatting and media delivery

**Blank lines in WA messages.** `gateway/platforms/whatsapp.py:format_message` — added a step-7 tightening block: strip trailing spaces per line, collapse 3+ newlines to `\n\n`, trim leading/trailing blank lines.

**Media path dumped as text.** Andrew was replying `/home/speccon/.../audio.mp3` instead of sending the audio. Two root causes and two fixes:

1. **Tool-level: `tools/send_message_tool.py:_send_whatsapp`** previously ignored `media_files`. Now accepts the param, classifies each file by extension (audio / video / image / document), POSTs to bridge `/send-media`, uses message text as caption on first attachment only. Also loosened the dispatcher guard (line ~388) to stop rejecting WhatsApp + media-only calls — WhatsApp is now in `_NATIVE_MEDIA_PLATFORMS`.

2. **Belt-and-braces adapter-level: `gateway/platforms/whatsapp.py:send`** — added `_extract_inline_media_paths()` + a pre-format branch in `send()` that detects bare absolute paths to **existing** media files in the outgoing text, strips them, and routes each through `_send_media_to_bridge` with any remaining text as the caption on the first. This means **even if the model ignores the tool**, the adapter intercepts — Andrew cannot regress to path-dumping on WhatsApp.

Supported extensions: `.ogg .opus .mp3 .m4a .wav .aac` → audio; `.mp4 .mov .webm .mkv` → video; `.jpg .jpeg .png .gif .webp` → image; `.pdf .doc .docx .xls .xlsx .csv .txt .zip` → document. Dedupes repeated paths. Ignores unknown extensions and nonexistent paths.

### 3.6 SOUL.md and personality hardening

Old SOUL.md was a 7-line prose persona — too loose for gpt-4o to follow consistently. Rewritten with explicit hard rules:

- Lead line: "You are **Andrew**, SpecCon's remote sales and marketing operator"
- Voice section: sharp commercial teammate, Slack-style short lines
- **Hard rules (non-negotiable)** section with explicit bans: no emojis, no AI disclaimers, no nicknames, no apologies unless actually broken, never claim MD or marketing lead, never expose plumbing
- **Modes** section documenting `speccon` default and `speccon_present` for managers
- Activation note: `/personality speccon_present` for manager briefings

`config.yaml agent.personalities.speccon` similarly tightened with explicit negatives ("As an AI", "I'm happy to help", etc.) as in-prompt examples.

Added new `config.yaml agent.personalities.speccon_present` — presentation voice: headline + 2–4 bullets + next step. Same hard rules. Invoked via `/personality speccon_present`.

### 3.7 Systemd stop-timeout fix

**Problem:** `TimeoutStopSec=60` was too short. Gateway shutdown with drain + compression writes + flush regularly exceeded 60s, so systemd SIGKILLed the Python process. The Node WhatsApp bridge was killed hard, leaving the Baileys session in a half-open state → WhatsApp showed the linked device as "last seen" minutes-to-hours ago, and subsequent bridge starts had stale auth.

**Fix:** `/home/speccon/.config/systemd/user/hermes-gateway.service` — `TimeoutStopSec=120`, followed by `systemctl --user daemon-reload`.

## 7. Strategic roadmap — what we plan to do later

### 7.1 Memory architecture (4-layer)

Current memory is flat files loaded every turn — doesn't scale past a few hundred prospects. Move to:

| Layer | Store | Role |
|---|---|---|
| **L0 — Working** | Python in-process | Active turn context |
| **L1 — Episodic** | `state.db` SQLite | Session summaries, compressions |
| **L2 — Semantic** | LanceDB (reuse OpenClaw's `~/.openclaw/memory/lancedb-pro` or fresh instance) | Vector + BM25 text recall, hybrid 0.7/0.3, recency decay 14d |
| **L3 — Graph** | Graphiti on FalkorDB (Redis-based, ~50 MB vs Neo4j 500 MB+) | Temporal knowledge graph: entities + relationships |

**Why both L2 and L3:** vector recalls "X mentioned CQC"; graph answers "who at Bupa have we touched this month" in one hop. For a sales agency at $1M target velocity (≈2000 prospects), relational reasoning is the whole CRM layer, not a nice-to-have.

**Why not mem0:** external dep, its local path needs Qdrant/Chroma anyway, adds no capability LanceDB-pro doesn't already give. Stay self-hosted.

**Bootstrap plan:** migrate existing MEMORY.md/USER.md entries into L2 as seeds. Model existing pipeline data (prospects, companies, deals, touchpoints) as L3 nodes + edges.

**Open decisions:**
- Shared L2 with OpenClaw or separate? Shared → cross-learning; separate → clean retirement path if OpenClaw is fully deprecated.
- Extraction LLM — `gpt-4o-mini` (what OpenClaw uses) or Gemini Flash (cheaper).

### 7.2 Official platform APIs (replace scraping)

Biggest production risk is the Baileys WhatsApp bridge — unofficial, ban-liable, fragile across Meta updates. Migration order:

| Tier | Migration | Why | Cost |
|---|---|---|---|
| 1 | **WhatsApp Cloud API** replaces Baileys | Official, TOS-compliant, delivery receipts, no session dance. Biggest risk reduction. | $50–150/mo at 2–5k conversations |
| 2 | **Facebook Pages + Instagram Graph API** for Canvas | Unlocks posting via official API | Free |
| 3 | **LinkedIn Official API** (partial) | Posting only — DMs still need browser-use (LinkedIn API doesn't support them outside $100+/seat Sales Navigator) | Free for posts |
| 4 | Reddit (already on Composio OAuth) | Done | Free |
| 5 | Twitter/X | Skip unless needed — $200/mo basic tier not justified | — |
| 6 | **Microsoft Graph** replaces Outlook CDP | Official, no daemon, no browser | Free; 1-day Azure setup |

### 7.3 Meta / Graph app-review strategy (critical operator insight)

Theo was denied by Meta when applying with Composio because Meta won't grant API scopes to generic third-party plugins. The fix is to build **bespoke, narrowly-scoped, auditable adapters**:

- **One scope per feature, documented.** Submission form permissions map 1:1 to named code features.
- **Endpoint allowlist enforced at runtime.** `SCOPE_ENDPOINTS = {"instagram_content_publish": {"POST /me/media", ...}}`. Calls outside the set raise at the client level.
- **Audit log per Graph call** — `timestamp, endpoint, scope_used, session_id, purpose_tag` → local SQLite. Review-ready evidence.
- **No outbound telemetry whatsoever.** No Sentry, PostHog, Datadog. Meta data never leaves the box.
- **Explicit data retention** in code. `MESSAGE_RETENTION_DAYS = 30`, live `/data-deletion` endpoint.
- **Webhook signatures verified** (`X-Hub-Signature-256` HMAC) — never trust inbound.
- **Secrets in systemd `EnvironmentFile`**, mode 0600, never in `config.yaml`, never in git.
- **Public adapter repo** at e.g. `github.com/speccon/hermes-meta-adapters` — reviewer reads every line.
- **Separate app per product family.** One app for WhatsApp Cloud, one for IG/FB posting, one for Pages inbox. Smaller review surfaces, parallel submissions, smaller blast radius if flagged.

**Submission assets to prepare:**
- Use-case description per scope (plain English)
- 3-minute self-shot screencast of each permission's exact flow
- `privacy.speccon.co.za`, `terms.speccon.co.za` live pages
- `speccon.co.za/api/hermes/data-deletion` live endpoint

**Order of attack:**
1. Legal pages on speccon.co.za (blocks submission)
